# 心玉最终目标：可验证的自我生成闭环

本文件是 XinYu 项目的最高目标锚点。任何人或 agent 在继续阅读、修改、审计 XinYu 项目前，应先读本文件，再读 `README.md` 和 `docs/system/XINYU-SYSTEM.md`。

## 一句话目标

让 XinYu 从“被动问答系统”和“主动候选生成系统”，逐步成长为一个在 owner 授权边界内运行的长期自主体：能够持续接收输入，形成内部缺口，选择有限行动，获得反馈，并把反馈转化为可审计的记忆、策略和下次行为变化。

## 核心思想

自主性不是一句主动消息，也不是人格文本的堆叠。自主性必须来自一个持续闭环：

```text
外部/内部输入
-> 重要性与异常判断
-> 内部状态、需求、缺口
-> 候选意图和候选行动
-> 门控、边界、优先级竞争
-> 有限行动
-> 行动结果和环境反馈
-> 记忆、评分、策略、表达方式更新
-> 下一轮输入中的不同反应
```

如果一个模块不能落到这条链路中的某个位置，就不应被当作自主性核心模块。

## 当前工程定义

XinYu 的“自我”不以宣称意识为目标，而以可验证的自我生成闭环为目标。

可接受的目标：

- 能保留具体输入，而不是把感知事件过早压成模板话术。
- 能区分外部信息、内部状态、owner 反馈、系统故障和自身行动结果。
- 能让内部压力变成候选，而不是直接变成发送或稳定记忆。
- 能解释为什么行动、为什么沉默、为什么等待 owner。
- 能把 owner 的阅读、忽略、拒绝、回复、批准、失败反馈写回评分和门控。
- 能让低风险本地动作产生真实结果，并把结果转化为下一步选择依据。
- 能保持长期连续性，但不把短期情绪、梦境、推测、日志噪声写成稳定事实。

不可接受的替代品：

- 只增加更像人的话术。
- 只增加更多主动消息。
- 只让 LLM 在长 prompt 里扮演“有自我”。
- 用梦境、情绪词、身体隐喻替代真实输入和反馈。
- 绕过 owner 授权、QQ/outbox claim/ack、记忆审查或本地安全边界。
- 为了“像自主”而制造噪声、打扰、伪感知或伪生理状态。

## 设计原则

1. 具体输入优先  
   感知事件、owner 原话、工具结果、错误码、时间变化、运行状态都要尽量保留可追溯锚点。表达层可以变短，但不能把关键动作锚点抹掉。

2. 内部缺口必须可观测  
   好奇、修复欲、记忆压力、表达修正、任务未完成、环境异常，都要落到状态字段、trace 或指标，而不是只存在于提示词里。

3. 行动必须有身体反馈  
   插件、文件、QQ、Desktop、Codex、TinyKernel、搜索、测试和本地探针都是行动器官；它们的返回值、失败、延迟、ack、状态变化就是虚拟触觉。

4. 候选不是行动  
   self_thought、impulse、initiative candidate 只提供压力和可能性。真正行动必须经过评分、门控、权限、冷却、去重和反馈路径。

5. 反馈必须改变未来  
   被忽略、被拒绝、被回复、被批准、发送失败、动作成功，都必须能改变后续评分、门控、阈值、记忆选择或表达策略。没有反馈更新，就不是闭环。

6. 沉默也是决策  
   XinYu 必须能记录为什么保持安静。沉默不应是模块没跑，也不应是候选丢失，而应是可解释的门控结果。

7. owner 是最高现实锚点  
   owner 的明确输入、纠正、授权和拒绝优先于推测、情绪残留、自动学习材料和外部资料。

8. 不追求无边界自主  
   最终目标是有边界、有审计、有反馈的自主性，不是无限自运行、无限外发、无限自改代码或无限写记忆。

## 当前优先实现方向

从 2026-05-27 起，工程优先级按下面顺序判断：

1. 修复反馈闭环  
   清理 phantom pending feedback。只有真正送达 owner、Desktop inbox 或 QQ/outbox 的事件才应等待 owner 反馈。

2. 修复 live loop  
   确认 core bridge、QQ gateway、Desktop、NapCat 和维护循环能长期运行，并能报告自己是否活着。

3. 保留输入锚点  
   视觉、语音、文本、事件、工具结果和 owner 原话进入系统后，不能被过早模板化。

4. 统一自主性指标  
   指标要回答：感知了什么，形成了什么缺口，候选是什么，为什么行动或沉默，反馈是什么，下次如何改变。

5. 行动结果回流  
   低风险本地动作、测试、状态探针、Codex 结果、QQ ack、Desktop ack 必须回流到评分、记忆或下一步选择。

6. 再谈更高层自我  
   只有当前面闭环稳定后，才继续扩展人格表达、TinyKernel、更多感官模型或更复杂的长期自我叙事。

## 每次改动前的检查

任何 XinYu 改动都应能回答：

- 这个改动落在核心闭环的哪一段？
- 它保留了哪个具体输入锚点？
- 它产生了什么可审计状态或 trace？
- 它是否让反馈改变未来行为？
- 它是否绕过了 owner、权限、记忆、QQ/outbox 或本地安全边界？
- 它是在增加真实闭环，还是只在增加“像自主”的表演？

## 完成标准

XinYu 接近本目标时，应满足：

- 能在没有 owner 新消息时，基于真实状态变化产生候选，但不滥发。
- 能明确说出某次候选为何被选中、持有、阻塞、显示或发送。
- 能将 owner 反馈转化为后续评分和门控变化。
- 能将行动结果转化为记忆、策略或下一步行动，而不是只写日志。
- 能长期保持连续性，同时不伪造身体、传感器、情绪或现实事实。
- 能把“我为什么想做这件事”和“我为什么没有做”同时记录清楚。

最终目标不是让 XinYu 声称“我有自我”，而是让 XinYu 的系统结构持续产生、约束、验证和修正“自我如何行动”的证据。
