# 外部项目借鉴登记

本文件记录 XinYu 项目在长期建设中参考、借鉴或对照过的外部项目。它用于：

- 保留未来鸣谢所需的仓库地址和作者信息。
- 避免重复调研同一项目时丢失上下文。
- 区分“设计思路借鉴”和“代码引入”。
- 跟踪许可证风险，尤其是 AGPL/GPL 等强 copyleft 项目。
- 将每次借鉴映射回 `docs/system/心玉自我生成长期路线大纲.md` 的阶段。

## 登记规则

每当 XinYu 因外部项目产生设计、架构、测试、交互或实现方向上的借鉴时，都必须在本文件新增或更新记录。

最低字段：

```text
项目名称：
仓库地址：
许可证：
首次登记时间：
参考状态：
对应路线阶段：
借鉴内容：
不采用内容：
落地方式：
验证信号：
代码引入：
鸣谢备注：
```

状态说明：

- `观察中`：已发现项目，尚未形成明确借鉴。
- `设计借鉴`：只学习架构、问题分解、接口形态、测试方式或产品组织方式。
- `实现重写`：按 XinYu 现有边界重新实现了类似能力，没有复制外部代码。
- `代码引入`：实际引入了外部代码、片段、资源或依赖。必须先完成许可证检查。
- `暂不采用`：看过后决定不采用，但保留原因。

许可证规则：

- AGPL/GPL 项目默认只能作为设计借鉴，不能直接复制代码进入 XinYu。
- MIT/Apache/BSD 等宽松许可证项目，也必须登记来源和具体引入范围。
- 不清楚许可证时，按高风险处理，先登记为 `观察中` 或 `设计借鉴`。

- 新增或修改模块必须产生状态字段、trace、报告、测试或 owner-visible 指标。
- 任何外发或自行动作必须仍经过 XinYu 的 gate、ack/drop、冷却、审批和反馈回流。

代码引入：无。不得直接复制该仓库代码，除非后续单独完成许可证评估。  
鸣谢备注：未来公开鸣谢时应标注为“设计参考/工程启发”，而非代码来源。
