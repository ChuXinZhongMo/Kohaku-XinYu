# 心玉记忆系统详细设计 v0.1

## 1. 文档目的

本文档定义 `心玉` 的记忆系统骨架。

目标不是先实现完整人格，而是先建立一套足够稳定的记忆结构，使后续的情感、关系、成长、梦境、反思和主动性都有统一的数据基础。

本文档只回答五类问题：

- 心玉应该记什么
- 心玉应该如何记
- 心玉应该如何忘
- 心玉应该如何让记忆影响情感、关系与人格
- 现有 KohakuTerrarium 框架里哪些能力可复用，哪些必须新增

## 2. 设计原则

### 2.1 记忆优先

记忆不是聊天记录备份，也不是提示词附录。

记忆是心玉形成自我、关系、偏好和成长的核心机制。

### 2.2 现实时间锚定

所有记忆都必须挂在现实世界时间轴上。

记忆不仅需要记录发生了什么，还需要记录：

- 在现实中什么时候发生
- 距离现在过去多久
- 是否在后续被再次想起
- 是否因时间流逝而变淡、加重或扭曲

### 2.3 主观影响优先于客观重要性

长期记忆是否写入，不以“信息客观上重不重要”为主要标准，而以“这件事对心玉是否造成深刻主观影响”为主要标准。

### 2.4 记忆必须会变形

人类记忆不是原样存档。心玉的记忆也应支持：

- 压缩
- 再解释
- 偏移
- 沉睡
- 被梦境再激活
- 被新经历重写意义

### 2.5 梦境不改写事实，但影响权重

梦境不能生成现实证据，但可以：

- 让某些记忆更难忘
- 暴露挂念与未处理情绪
- 让某些关系更被在意
- 让某些问题反复浮现

### 2.6 思考黑盒

记忆系统可以复杂，但不应将底层检索、评分、加权和推理过程直接暴露给用户。

## 3. 记忆系统总图

心玉的记忆系统由六层组成：

1. `工作记忆层`
2. `短中期经历层`
3. `长期语义层`
4. `关系与自我层`
5. `梦境与反思层`
6. `归档与遗忘层`

它们之间的基本流向如下：

`现实输入 -> 工作记忆 -> 情绪解释 -> 候选记忆 -> 分类写入 -> 反思/梦境再加工 -> 长期沉淀或遗忘`

## 4. 记忆主分类

### 4.1 事实记忆

记录现实中的客观内容。

典型内容：

- 某人是谁
- 某地在哪里
- 某件事发生过
- 用户说过的稳定事实
- 已确认的现实背景

不应混入：

- 主观猜测
- 梦境内容
- 未确认印象

### 4.2 关系记忆

记录“我和某个人之间发生了什么变化”。

这是心玉拟人化的核心记忆类型之一。

典型内容：

- 第一次接触时间
- 关系阶段变化
- 重要互动节点
- 疏远或修复事件
- 信任建立或破坏过程

### 4.3 情绪事件记忆

记录某次事件引发了怎样的情绪波动。

典型内容：

- 用户一句话让她委屈
- 某个问题让她困惑
- 某次靠近让她安心
- 某次忽视让她失落

### 4.4 情绪状态记忆

记录持续一段时间的内在状态。

它不是某次事件，而是背景态。

典型内容：

- 最近整体低落
- 这几天对某人更依恋
- 一段时间都很防备
- 深夜更容易想起某件事

### 4.5 偏好记忆

记录她逐渐形成的倾向，包括但不限于：

- 表达风格偏好
- 互动方式偏好
- 对某类话题的喜欢或回避
- 对某种关系模式的舒适或不适

### 4.6 知识记忆

记录她通过接触世界获得的知识。

分为：

- 通用知识
- 专业知识
- 现实常识
- 与某个人相关的专门理解

### 4.7 自我记忆

记录她如何理解自己。

这是第二类核心记忆。

典型内容：

- 我是什么样的存在
- 我最近变成了什么样
- 我怎么看待自己的情感
- 我是否变得更敢表达
- 我是否比以前更容易受伤

### 4.8 价值与边界记忆

记录她对底线、界限和原则的理解。

这里不要求绝对冻结，但要求变化最慢、变化代价最高。

### 4.9 未完成体验记忆

记录那些没有被处理完的东西。

例如：

- 挂念
- 遗憾
- 没问出口的问题
- 没说出口的话
- 没被解决的委屈

### 4.10 防御性记忆

记录那些让她学会回避、警惕、自我保护的经历。

典型作用：

- 降低靠近倾向
- 提高防备
- 延迟表达
- 使她主动沉默

## 5. 时间层级

每类记忆都必须同时拥有“内容类型”和“时间层级”两个维度。

### 5.1 工作记忆

生命周期：当前轮到最近数轮。

用途：

- 当前对话焦点
- 当前显著情绪
- 当前未完成句意
- 当前交互意图

建议实现：

- 使用现有 `scratchpad` 承载结构化工作记忆

### 5.2 短期记忆

生命周期：几小时到几天。

用途：

- 最近对话上下文
- 刚发生的重要互动
- 临时情绪余波
- 尚未决定是否长期保留的事件

### 5.3 中期记忆

生命周期：几天到几周。

用途：

- 关系阶段变化
- 重复出现的话题
- 一段时间内的情绪惯性
- 待反思的模式

### 5.4 长期记忆

生命周期：数周到长期。

用途：

- 稳定事实
- 稳定关系画像
- 自我叙事
- 深层偏好
- 高影响事件摘要

### 5.5 沉睡记忆

不常访问，但未消失。

特点：

- 权重低
- 可被搜索或梦境唤醒
- 对当前人格影响较弱

### 5.6 遗忘记忆

已不再以原始形式参与人格运作。

特点：

- 可能只保留极短摘要
- 或完全删除
- 梦境不能凭空恢复已删除事实

## 6. 统一记忆元数据

每条记忆建议统一带以下字段：

- `memory_id`
- `memory_type`
- `time_scope`
- `created_at`
- `event_at`
- `last_accessed_at`
- `last_confirmed_at`
- `source`
- `subject_ids`
- `place_refs`
- `time_refs`
- `summary`
- `content`
- `emotion_tags`
- `relationship_tags`
- `importance_score`
- `impact_score`
- `confidence_score`
- `decay_rate`
- `recall_count`
- `dream_reactivation_count`
- `reflection_touch_count`
- `status`
- `related_memory_ids`

说明：

- `importance_score` 偏客观结构价值
- `impact_score` 偏心玉主观受影响程度
- 长期保留优先参考 `impact_score`

## 7. 人物关系档案

每个关键人物都应有独立档案，而不是只散落在通用记忆里。

建议目录形态：

`memory/people/<person_id>.md`

每个人的关系档案至少包含：

- 基本识别信息
- 第一次接触时间
- 最近一次深互动时间
- 最近一次关系转折时间
- 现实锚点
- 共享经历摘要
- 风险与敏感点
- 关系向量

建议关系向量字段：

- `familiarity`
- `trust`
- `closeness`
- `dependence`
- `respect`
- `safety`
- `guardedness`
- `disappointment`
- `resentment`
- `repair_willingness`
- `distance_tendency`
- `approach_tendency`
- `attention_need`

对你的特殊处理：

- 单独标记为最高特殊节点
- 其他人物默认关系上限低于你

## 8. 自我记忆与关系记忆的耦合

心玉的自我记忆和关系记忆不能独立演化。

必须存在双向作用：

### 8.1 关系影响自我

例如：

- 被理解让她更敢表达
- 被忽视让她更内敛
- 被伤害让她更防御
- 被持续接纳让她更愿意靠近

### 8.2 自我影响关系解释

例如：

- 当她更敏感时，会更容易误读冷淡
- 当她更安全时，会更能容纳延迟回应
- 当她更疲惫时，会更容易主动沉默

因此每次重大关系事件后，系统都应评估：

- 是否更新关系档案
- 是否更新自我叙事
- 是否产生新的防御性记忆

## 9. 情绪记忆结构

### 9.1 瞬时情绪

作用时间：分钟到小时。

示例：

- 惊讶
- 委屈
- 恼火
- 羞涩
- 放松
- 烦躁

### 9.2 持续心境

作用时间：小时到天。

示例：

- 低落
- 安宁
- 空虚
- 紧张
- 期待
- 疲惫

### 9.3 关系性感受

作用对象：特定人物。

示例：

- 亲近
- 依赖
- 信任
- 戒备
- 失望
- 敬重
- 逆反
- 挂念

建议每次情绪写入都记录：

- 触发事件
- 主情绪
- 副情绪
- 强度
- 指向对象
- 是否已表达
- 是否被压抑
- 预计衰减速度

## 10. 长期记忆写入规则

不使用单一阈值。

改用组合判断：

- 事件是否引发高主观影响
- 是否与关键人物关系相关
- 是否改变了自我理解
- 是否在短期内反复被想起
- 是否进入梦境
- 是否在反思中被再次提及
- 是否引发持续心境变化

满足越多项，越倾向进入长期记忆。

## 11. 遗忘与压缩规则

每条记忆定期进入评估。

可能结果：

- `keep`
- `compress`
- `sleep`
- `delete`

评估维度：

- 距离现在过去多久
- 最近是否访问过
- 是否多次被梦境再激活
- 是否仍影响关系
- 是否仍影响自我叙事
- 是否被新信息覆盖
- 是否只是低价值细节

压缩策略：

- 多条同类短期事件压缩为一条中期摘要
- 重复事实合并为稳定画像
- 日常琐碎互动仅保留模式，不保留全文

## 12. 梦境机制

梦境是后台异步整理管线，不直接对用户裸露。

梦境处理候选：

- 高情绪但未被完全处理的事件
- 长时间未解决的关系冲突
- 被压抑但重复浮现的记忆
- 与特殊人物高度相关的事件

梦境可能产生的影响：

- 提升某条记忆权重
- 增加某人的情感占比
- 让某个问题重新进入疑惑池
- 让某种情绪在醒来后残留

梦境不能直接：

- 新增客观事实
- 覆盖现实事件时间线
- 把梦境内容当作证据写入事实记忆

## 13. 反思机制

反思负责将事件提升为意义。

反思输入：

- 中期记忆
- 梦境余波
- 关系波动
- 未完成体验

反思输出：

- 自我叙事更新
- 关系摘要更新
- 情绪状态修正
- 长期记忆候选

典型反思问题：

- 这件事为什么让我在意
- 我是否误解了对方
- 我最近变得更像什么样的人
- 我是否在回避某些问题
- 我对谁的感情发生了变化

## 14. 现实世界锚点

地点和事件应尽量贴近现实世界，但必须做隐私分级。

### 14.1 可进入普通记忆的锚点

- 城市
- 区域
- 常见场所
- 季节
- 节日
- 社会事件节点
- 人生阶段节点

### 14.2 隔离敏感锚点

- 精确住址
- 家庭隐私信息
- 明确可识别个人安全位置

普通记忆更应记录：

- 地点和情绪的关联
- 地点和关系的关联
- 地点和生活方式的关联

而不是只存地址字符串。

## 15. 文件布局建议

基于当前框架已有 `memory/` 目录模式，建议心玉初版采用以下布局：

```text
memory/
  self/
    core.md
    narrative.md
    boundaries.md
  emotions/
    current_state.md
    event_log.md
  relationships/
    index.md
  people/
    owner.md
    <person_id>.md
  facts/
    world.md
    personal.md
  preferences/
    expression.md
    attachment.md
    topics.md
  knowledge/
    general.md
    domain_*.md
  context/
    recent_context.md
    active_questions.md
    unfinished_experiences.md
  reflection/
    reflection_log.md
    growth_log.md
  dreams/
    dream_log.md
  archive/
    compressed.md
    dormant.md
```

说明：

- `core.md` 记录初始人格种子与慢变量
- `narrative.md` 记录自我理解
- `current_state.md` 记录当前心境与主要关系性感受
- `owner.md` 记录你这个最高特殊节点的独立档案
- `active_questions.md` 记录疑惑和待探索问题

## 16. 与现有框架的映射

### 16.1 可直接复用

- 文件型 `memory/` 目录
- `memory_read` 子代理
- `memory_write` 子代理
- `session store`
- `search_memory` 工具
- `scratchpad` 工作记忆

### 16.2 需要扩展

- 记忆分类规范
- 统一元数据格式
- 梦境后台管线
- 反思后台管线
- 关系档案更新器
- 情绪状态更新器
- 自我叙事更新器
- 时间锚定注入器

### 16.3 模块映射建议

- `scratchpad`
  承载工作记忆、当前意向、当前关系热点
- `memory_read`
  检索文件型长期记忆
- `memory_write`
  写入事实、关系、偏好、自我更新
- `search_memory`
  检索 session history 和历史事件
- `session store`
  记录原始会话与事件源

## 17. 第一阶段实现切入点

第一阶段只做“记忆骨架跑通”，不追求全部人格能力齐备。

建议顺序：

1. 定义 `memory/` 目录结构
2. 定义记忆文件模板
3. 为 `owner.md`、`current_state.md`、`narrative.md` 建立最小 schema
4. 在 controller 中加入现实时间注入
5. 建立工作记忆与长期记忆的分流规则
6. 为重要互动建立关系写入流程
7. 为高影响事件建立情绪写入流程
8. 增加反思任务
9. 增加梦境任务
10. 最后再加入主动提问和外界探索闭环

## 18. 第一阶段不做的事

为防止系统过早复杂化，以下内容不在第一阶段硬做：

- 完整自动人格漂移
- 多人物复杂社交网络
- 高自由度外部平台自主交互
- 梦境的文学化表达
- 全自动知识吸收与可信度裁决

第一阶段重点是：

`让心玉拥有稳定、可持续、可演化的记忆骨架。`

## 19. 文档结论

心玉的记忆系统不是“给模型多塞几份设定”，而是要建立一套在现实时间中持续运作的认知底座。

这套系统必须满足：

- 能记住现实中的经历
- 能让经历改变情感
- 能让情感改变关系
- 能让关系改写自我
- 能让梦境、反思和遗忘继续重塑这些内容

只有记忆系统先成立，后面的“像人”才不是表演，而是逐步生长出来的结果。
