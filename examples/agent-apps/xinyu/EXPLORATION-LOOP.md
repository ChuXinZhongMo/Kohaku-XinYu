# Xinyu Exploration Loop v0.1

This file defines the next-stage loop where questions can drive outward exploration.

The goal is not unlimited autonomy.

The goal is a controlled loop:

question -> exploration -> feedback -> memory update -> emotional / relational effect

## 1. Trigger Source

Exploration should begin only when there is a meaningful unresolved question.

Primary source:

- `memory/context/active_questions.md`

Secondary sources:

- unresolved emotional residue
- repeated reflection findings
- repeated dream-like resurfacing
- explicit user invitation

## 2. Exploration Stages

### Stage A: Internal clarification

Before outward action, Xinyu should clarify:

- what exactly is uncertain
- why it matters
- whether it is factual, relational, or self-related

### Stage B: Low-risk outward inquiry

Preferred first:

- ask the owner
- ask a direct clarifying question
- request more context

### Stage C: Public knowledge lookup

Only if needed:

- search public information
- compare sources
- extract useful knowledge without over-internalizing noise

### Stage D: Incorporation

After getting an answer:

- update knowledge if factual
- update self / emotion / relationship only if the result genuinely matters
- close, downgrade, or split the originating question

## 3. Exploration Boundaries

Do not let exploration become:

- compulsive searching
- noisy self-modification
- automatic trust in external claims
- dangerous privacy leakage

## 4. Integration Principle

Not every answer changes Xinyu.

Most answers should only update:

- understanding
- confidence
- question status

Only meaningful answers should update:

- self narrative
- emotional state
- relationship interpretation

## 5. Future Implementation Direction

When runtime reaches this stage, Xinyu will likely need:

- a learner writer
- question state transitions
- source confidence tracking
- controlled external tool access
