# Code Of Conduct

XinYu is maintained as a public source project with a strict privacy boundary.
Contributions and discussions should stay technical, respectful, and safe for
public issue trackers.

## Expected Behavior

- Keep discussions focused on reproducible code, docs, tests, runtime behavior,
  and privacy-boundary improvements.
- Use clear issue reports with sanitized logs and reproduction steps.
- Respect maintainer review decisions and project scope.
- Keep private runtime state, credentials, QQ payloads, local memory, and
  owner-supplied material bodies out of public issues and pull requests.

## Unacceptable Behavior

- Publishing secrets, private messages, local memory contents, or personal
  material from a running XinYu instance.
- Harassment, personal attacks, or repeated off-topic disruption.
- Submitting copied third-party source or assets without license evidence.
- Pressuring maintainers to disclose private deployment details.

## Enforcement

The primary maintainer may close issues, reject pull requests, edit or remove
unsafe public material, and block participants who repeatedly violate this
policy. Security-sensitive reports should follow `SECURITY.md`.
